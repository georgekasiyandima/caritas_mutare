# Caritas Mutare — Project documentation

This folder holds the official architecture and design documentation for the Caritas Mutare web application.

| Document | Purpose |
|----------|---------|
| **[ARCHITECTURE.md](ARCHITECTURE.md)** | Technical architecture, principles (avoid tight coupling, composition over inheritance, AHA, separate concerns, derive state, least surprise), craft guidelines, debugging/resilience, testing, and developer experience. |
| **[DESIGN_AND_CONTENT_PLAN.md](DESIGN_AND_CONTENT_PLAN.md)** | Design system (typography, colours, spacing), how we represent projects and programmes, donor/partner trust, NGO best practices, and asset folder structure. |
| **[NEXT_STEPS.md](NEXT_STEPS.md)** | Recommended next steps: using assets in the UI, partner strip, programme pages, leadership, fonts, routes, content, and deploy. |

These docs are the single source of truth for how we build and maintain the platform. Update them when we change direction or add new conventions.
